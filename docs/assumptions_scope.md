## Assumptions & Scope

### Assumptions
* **Context:** This is an **internal university system**, not a public commercial product.
* **Expected Users:**
    * Hundreds of students browsing events.
    * Dozens of event organizers submitting events.
    * A small number of staff administrators.
* **Performance:** Concurrent usage is **low to moderate**. There are no strict real-time or high-availability requirements.
* **Resources:**
    * The system is developed and maintained by a **small team (2â€“4 engineers)**.
    * Cloud hosting is available, but **budget and operational resources are limited**.

### Out of Scope
The following features are explicitly excluded from the current phase:
* Native mobile applications
* Public APIs for external partners
* Real-time chat or live streaming
* Complex role-based permission systems
* Advanced analytics or recommendation engines
